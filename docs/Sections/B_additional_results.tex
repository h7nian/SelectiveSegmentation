\section{Additional experimental results and reproducibility}
\label{app:additional-results}

The final tables are generated from one validated analysis object; no row is
selected by its observed AURC, interval, or tail probability. This source
intentionally contains no transcribed result values.

\subsection{Immutable campaign and execution graph}

The reproducibility release is accepted only when it contains exactly sixteen
schema-v2 frozen-map manifests, one for every declared model--dataset condition.
Each manifest binds the ordered sample identifiers, probability/truth payload,
cohort and checkpoint metadata, evaluator-source fingerprint, package versions,
device metadata, and SHA-256 digests. A no-overwrite campaign lock binds those
sixteen manifests to \(\gamma=0.5\), midpoint \(M\in\{2,8,32\}\), and seed 0.
Append-only, phase-specific submission receipts record every intended Slurm job
and its returned job identifier; unresolved submission intent blocks blind
resubmission.

The execution graph separates expensive inference from repeated geometry. One
GPU \emph{freeze} job per condition writes the immutable native-resolution map
artifact; every freeze job is eligible on either of two private A100 partitions. One CPU
\emph{common} job per artifact computes the three observed risks, both raw-pixel
boundary auxiliaries, and the eight \(M\)-independent scores. Finally, each
independent pair \((\text{condition},M)\) is one CPU simulation job---48 jobs,
not a Slurm array---and writes only the Dice, nHD, and nHD95 scores for that
\(M\). A strict assembler joins one common shard and the three \(M\)-specific
shards only when schemas, identities, row counts, sample ordering, hashes, and
campaign bindings agree exactly. This design computes each shared floating-point
quantity once and lets the three numerical resolutions run in parallel.

The manifests, lock, assembled records, diagnostics, and analysis JSON are the
authoritative computational provenance. They retain checkpoint and source
hashes plus software/hardware versions; a portable release descriptor adds the
exact cached CLIPSeg revision, torchvision weight digest, and available
training-configuration hashes without exporting private execution paths.
