\section{Additional experimental results and reproducibility}
\label{app:additional-results}

The six core outcome tables are generated from one validated analysis object;
each auxiliary table is generated from its own strictly validated analysis
artifact. No row is selected by its observed AURC, interval, or tail
probability. This source intentionally contains no transcribed result values.

\subsection{Immutable campaign and execution graph}

The anonymous post-inference release is accepted only when it contains exactly
sixteen assembled manifest/record pairs, one for every declared
model--dataset condition, together with the immutable campaign lock, analysis,
and generated tables. Each path-free assembled manifest binds the condition,
ordered sample identities, row count, schema, source digest, and record
SHA-256. The released per-image records are sufficient to recompute every AURC,
contrast, interval, and table. Probability maps, reference masks, checkpoints,
submission receipts, job identifiers, and machine metadata are deliberately
excluded from the anonymous package.

The separately retained local execution audit binds sixteen schema-v3 frozen
map manifests through a no-overwrite schema-v2 campaign lock. Its graph
separates expensive inference from repeated geometry: one GPU \emph{freeze}
job per condition writes the native-resolution map artifact; one CPU
\emph{common} job computes observed risks, raw-pixel boundary auxiliaries, and
the \(M\)-independent scores; and each independent
\((\text{condition},M)\) pair is one CPU score job---48 jobs, not a Slurm
array. A strict assembler joins one common shard and the three \(M\)-specific
shards only when schemas, identities, counts, sample ordering, hashes, and lock
bindings agree. Append-only phase receipts support identity-checked
reconciliation and block blind resubmission. These private execution records
validate the pipeline but are not part of the anonymous release.

For anonymous statistical reproduction, the assembled records, campaign lock,
analysis JSON, and generated tables are authoritative. For local inference and
scheduler forensics, the frozen manifests, scientific-input root, diagnostics,
receipts, and schema-v2 replay lock provide the additional private provenance.
The two closures are checked separately rather than implying that private maps
or scheduler records are distributed to reviewers.
